How should the display type scale down?

The hero headline is 160px Anton, post titles are 144px. At 390px wide, these need to shrink significantly. Three philosophies:

A — Fluid with clamp()

Type scales smoothly between a min and max using viewport units. No hard breakpoints — always proportional. 160px → ~52px at 390px.

Pros

  • Buttery smooth at every width
  • No magic numbers per breakpoint

Cons

  • Less editorial control
  • In-between sizes can look awkward

& THE
UNSEEN
mobile · 64px (was 160px)

B — Fixed steps per breakpoint

Deliberate type sizes locked to each breakpoint. 160px (desktop) → 96px (tablet) → 64px (mobile). Editorial rhythm is intentional at every size.

Pros

  • Designer controls every step
  • Feels intentional & editorial

Cons

  • More breakpoint rules to maintain
DESIGN & THE UNSEEN
mobile · always fills container width

C — Full-width display type

Headlines always fill the container using font-size: clamp() or vw tuned so text bleeds edge-to-edge. Very brutalist — the type IS the layout.

Pros

  • Maximum brutalist impact
  • Works at any width automatically

Cons

  • Multi-word headlines can break awkwardly
  • Readability varies by title length
